# antd-more

基于 `antd@4` 扩展的组件


## 贡献

### commit 规范

**格式: 冒号后面有空格**

```shell
<type>: <subject>
```

- **upd** - 更新某功能（不是 feat, 不是 fix）
- **feat** - 新功能（feature）
- **fix** - 修补bug
- **docs** - 文档（documentation）
- **style** -  格式（不影响代码运行的变动）
- **refactor** - 重构（即不是新增功能，也不是修改bug的代码变动）
- **test** - 增加测试
- **chore** - 构建过程或辅助工具的变动

示例

```shell
git commit -m 'feat: 增加 xxx 功能'
git commit -m 'bug: 修复 xxx 功能'
git commit -m 'upd: 更新版本号'
```
